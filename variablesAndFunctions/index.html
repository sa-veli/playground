Looks like we place variables inside :root pseudo-class.

Syntax and convinient way to organize styles

:root {
/* FONT */
--FF: "Nunito" sans-serif;
--FS: 1.5rem;
--FS-HEADING: 4.5rem;

/* COLOR */
--VARIABLENAME: #000;
--ALT-BGCOLOR: yellow;

/* GENERAL */
--PADDING: 0.5rem;
}

background-color: var(--VARIABLENAME);
background-color: var(--ALT-BGCOLOR);
font: var(--FS) var(--FF); Notice absence of comma between vars.

Notes

It's ok to use one variable value on two or more variables.
For example:
--BGCOLOR: white;
--LIGHT-COLOR: white;

You can define variable using another variable

You can redefine variable value. For example, redifine value for one of the classes.

.green-color { --BGCOLOR: green; }

Use vh (not vw) units to define font size. The code on the image translates to:

Define variable "FS". Variable value is min function which chooses minimum of two values: 2.25rem and 3vh.

It's handy when you can set one biggest font and need to change it when viewport shrinks. It doesn't scales bigger than 2.25rem.
It is useful to set padding and/or margin to element to which you applied this function to em units. That way padding and/or margin scales with font size.

There is also max() function. You can think by yourself how it can be used to set up font size.

font-size: clamp(1.75rem, 3vh, 2.25rem); Clamp function chooses between three values. Second value is best, first is min, third is max.

There is calc() function. See MDN for description.
